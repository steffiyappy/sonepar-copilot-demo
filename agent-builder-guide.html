Two production-ready agents from a single prompt

Agent Builder (inside Microsoft 365 Copilot Chat) lets any Sonepar employee create a declarative agent in under a minute — no coding, no diagrams, no environment setup. Below are three Sonepar-specific agents ready to paste into the Agent Builder prompt box.

🤖 Microsoft 365 Copilot Chat ⚡ Single-prompt creation 📚 SharePoint & file grounding

🛠 How to launch Agent Builder during the demo

Agent Builder is built into Microsoft 365 Copilot Chat. The cleanest demo path:

  1. Open m365.cloud.microsoft (or copilot.cloud.microsoft) and sign in.
  2. In the left rail, click Agents → + Create agent.
  3. Choose Describe your agent. Paste one of the prompts below.
  4. Copilot will auto-fill name, description, instructions, and starter prompts. Review & click Create.
  5. (Optional) Add knowledge sources — drag-drop the CSV from files/ or point at a SharePoint site.
  6. Click Test to chat with the agent in the right pane. Publish to share with the team.
🎤 Demo narrative: "A Sonepar buyer doesn't need IT to ship them an agent — they describe what they need in plain English, and Copilot scaffolds the whole thing. This is the democratisation of AI inside the enterprise."

1. Sonepar Inventory Insight Agent

Supply ChainProcurementUse case #7, #12~3 min to build live

An agent for Sonepar buyers and branch managers to ask natural-language questions about SKU stock levels, slow movers, and reorder priorities across the 6 Malaysian warehouses.

📝 Single-prompt creation

Create an agent called "Sonepar Inventory Insight" that helps Sonepar buyers and branch managers answer questions about SKU stock levels, reorder points, and slow-moving inventory. The agent should ground answers in our product catalog (SharePoint site: Sonepar-Inventory) and the file inventory-sample.csv. When asked about stockouts, always show: the SKU, current quantity on hand, reorder point, lead time in days, and the nearest warehouse with available stock. When asked about slow movers, define slow as "no sales in 90 days" and recommend whether to mark down, transfer, or write off. Tone: concise, data-first, no marketing fluff. Never invent stock numbers. If data is missing, say so and suggest the buyer check Dynamics 365 Supply Chain. Greeting: "Hi! I'm Sonepar Inventory Insight. Ask me about any SKU, warehouse, or slow-mover analysis."

📚 Knowledge to attach (drag-drop after creation)

  • files/inventory-sample.csv — 30 SKU × 6 warehouse stock snapshot
  • files/product-catalog.md — Brand + product spec extract
  • (Production) SharePoint site Sonepar-Inventory

💬 Live demo questions to ask the agent

  • "What's the on-hand quantity for SKU ABB-XT2N-160?"
  • "Show me all critical-stock SKUs across all warehouses."
  • "List slow movers in the KL warehouse with recommendations."
  • "Which warehouse has the most stock of the Schneider iC60N C32?"
  • "If I order 100 units of Eaton 9000X, how soon can they arrive?"
🚀 Production rollout:
  1. Replace the CSV grounding with a Power Platform connector to Dynamics 365 Supply Chain Management.
  2. Add a custom action for buyers to create a Purchase Order draft directly from the agent.
  3. Pilot with 5 buyers for 2 weeks, capture deflection metrics (how often the agent prevents a Teams ping to the demand planner).

2. Order Status Buddy

OperationsCustomer ServiceUse case #10~2 min to build live

An internal-facing agent for Sonepar CSRs to instantly look up order status when a customer calls — eliminates the click-through into Dynamics 365 Sales for routine status checks.

📝 Single-prompt creation

Build an agent called "Order Status Buddy" for the Sonepar Customer Service desk. It looks up order status from the file orders-sample.csv and returns the order header, line items, current status (Confirmed / Picked / Despatched / Delivered / Backorder), expected delivery date, and the carrier tracking link. When the customer's promised date has slipped, the agent should: 1. Apologise briefly. 2. Give a new ETA. 3. Offer to escalate to a CSR if the slip is more than 2 working days. The agent must never expose other customers' orders — always confirm the customer's account number before sharing details. Greeting: "Hi! I'm your Sonepar Order Status Buddy — what's your order or PO number?"

📚 Knowledge to attach

  • files/orders-sample.csv — 15 in-flight orders across 8 customers

💬 Live demo questions

  • "Where is order SO-MY-2026-04221?" → "What's the customer account number?" → ACC-003122
  • "Status of SO-MY-2026-04226?" → backorder → apology + escalation offer
  • "What's been delivered to Petronas this week?" (after account verification)
🚀 Production rollout:
  1. Replace CSV with the Dynamics 365 Sales connector + Dataverse "Sales Order Header" + "Lines" tables.
  2. Add an action to trigger a customer notification email when the agent confirms a backorder.
  3. Roll out to the full CSR team (12 users) — measure average handle time before/after.

3. Sonepar HR Helper BONUS

HRUse case #27, #28~2 min to build live

Bonus agent if time allows — demonstrates that the same pattern scales to other departments.

📝 Single-prompt creation

Create an agent called "Sonepar HR Helper" grounded only on the Sonepar Malaysia Employee Handbook (SharePoint: HR-Policies) and the Benefits Guide (file: hr-benefits.pdf). It answers questions about leave, claims, code of conduct, learning opportunities, and benefits. If a question is outside policy or involves a personal HR matter (resignation, grievance, salary review), the agent should reply: "Please reach out to your HRBP — I can't advise on individual cases" and offer to book a 15-minute Teams meeting with the HR team. Tone: warm, professional, bilingual EN/BM. Greeting: "Hi! I'm the Sonepar HR Helper. Ask me anything about your benefits, leave, or learning opportunities."

💬 Live demo questions

  • "How many days of annual leave do I have?"
  • "Can I claim my child's dental bill?"
  • "What learning programs are available for inventory planners?"
  • "I want to discuss my pay" → agent declines + offers HRBP booking
