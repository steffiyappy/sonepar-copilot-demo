Build the Sonepar Customer Service Agent live

A production-ready agent that handles order tracking, stock availability, technical document Q&A, and intelligent escalation for Sonepar Malaysia's 7,000+ industrial customers — published to Microsoft Teams and the customer portal.

🛠 Microsoft Copilot Studio 📦 Dynamics 365 (action) 📚 SharePoint (knowledge) 💬 Teams + Web channels ✓ Use case #19

📑 Build sequence (8 steps · ~18 min on stage)

  1. Create the agent (name, description, instructions)
  2. Add Knowledge — Product catalog (SharePoint)
  3. Build Topic 1 — "Track my order"
  4. Build Topic 2 — "Check stock availability"
  5. Build Topic 3 — "Product spec Q&A" (generative)
  6. Build Topic 4 — "Escalate to a human" (Teams handoff)
  7. Add Action — LookupOrderStatus (Power Automate to Dynamics 365)
  8. Test, publish, and add channels (Teams + Web)
1

Create the agent

2 min · copilotstudio.microsoft.com
  1. Go to copilotstudio.microsoft.com and sign in with the demo tenant. Make sure the environment is set to Sonepar Demo (default) in the top-right.
  2. Click + Create → New agent.
  3. When asked "Describe your agent in your own words", paste the prompt below:
Create a customer service agent for Sonepar Malaysia (a Sonepar Company), the leading B2B electrical and industrial distributor in Malaysia. The agent serves 7,000+ industrial customers and helps them with four things: (1) tracking the status of their open orders, (2) checking stock availability across our 6 warehouses (KL-Main, Penang, JB, Kuching, Ipoh, Kuantan), (3) answering product specification and installation questions grounded in our product catalog, and (4) escalating complex or sensitive issues to a human Customer Service Representative via Microsoft Teams. The agent must verify the customer's account number before sharing any order details. It should be bilingual EN/BM. It must never quote prices — direct any pricing question to the customer's named account manager. Tone: concise, professional, helpful. Greeting: "Hi! I'm Sonepar Malaysia's Customer Service Agent. How can I help today?"

Copilot Studio will auto-suggest a name, description, and starter topics. Click Create.

💡 Stage tip: While the agent generates, narrate that Copilot Studio is auto-bootstrapping topics, instructions, and conversation starters from a single prompt — a massive lift versus the old "blank canvas" build.

Once created, open Overview and confirm/replace these values:

Sonepar Customer Service Agent

24/7 customer-facing agent for Sonepar Malaysia — order tracking, stock checks, product Q&A, and escalation.
Instructions

2

Add Knowledge — Product catalog

2 min · Knowledge tab
  1. Open the agent's Knowledge tab.
  2. Click + Add knowledge → SharePoint.
  3. Paste this SharePoint URL: https://soneparmy.sharepoint.com/sites/Sonepar-Inventory (or the demo tenant equivalent).
  4. Name: Sonepar Product Catalog. Description: 50,000+ product specs, datasheets, and installation guides for ABB, Schneider, Siemens, Philips, Legrand, Rockwell, Eaton, Wago, Prysmian, Hager, and Omron.
  5. Toggle Use generative answers ON for the agent overall (Settings → Generative AI → Generative orchestration).
📝 If demo tenant has no SharePoint: Upload files/product-catalog.md as a file knowledge source instead. Same effect, faster setup.
3

Topic 1 — "Track my order"

3 min · Topics tab

Copilot Studio will likely have auto-created a similar topic. Either edit it, or click + Add a topic → Create from description and paste:

Trigger this topic when the customer asks where their order is, asks for an ETA, status, tracking number, or mentions an order number (e.g. "SO-MY-2026-04221"). Ask for the customer's account number (e.g. ACC-001245) and the order number if not already provided. Then call the LookupOrderStatus action with those inputs. Present the result as: order date, current status, expected delivery date, warehouse, carrier, tracking link. If the order is in Backorder status, apologise, provide the new ETA, and offer to escalate. Never share order details for an account number the customer has not provided.

Trigger phrases (Copilot Studio infers, but verify these are present):

  • Where is my order
  • Order status
  • Track my order
  • ETA on my delivery
  • SO-MY (regex match for order number pattern)
4

Topic 2 — "Check stock availability"

3 min · Topics tab

Add another topic from description:

Trigger this topic when the customer asks whether something is in stock, available, or asks for quantity at a specific warehouse. Ask for the product (SKU or description like "ABB XT2N 160") and the preferred warehouse if not provided. Use the inventory knowledge source (or the CheckInventory action when available) to return: SKU, brand, on-hand qty, reorder point, warehouse, and lead time in days. If stock is below reorder point, mark as "Reorder advised" and proactively suggest the next-nearest warehouse with healthy stock. Never invent quantities — if data is missing, say so and offer to log a callback from the buyer.
💡 Demo-day shortcut: For the live demo we will ground this in the Product Catalog SharePoint knowledge + the inventory-sample.csv uploaded as a file knowledge source. In production, swap for a real Dynamics 365 Supply Chain action.
5

Topic 3 — "Product spec & technical Q&A"

2 min · Topics tab

This is the easiest topic — generative orchestration handles it automatically using the SharePoint product catalog knowledge source you added in Step 2. Add a lightweight topic:

Trigger when the customer asks for a datasheet, specification, installation guide, dimensions, ratings, hazardous-area suitability, or any product-spec question. Answer using the Sonepar Product Catalog knowledge source. Always cite the source document name. If the catalog does not have the answer, say "I don't have that in our published catalog — let me log a callback from the product specialist."

Test phrases to use live:

  • "What's the kA rating of the ABB XT2N 160?"
  • "Is the Eaton 9000X suitable for hazardous areas?"
  • "Send me the datasheet for the Schneider iC60N C32"
6

Topic 4 — "Escalate to a human"

2 min · Topics tab

Add the escalation topic:

Trigger this topic when the customer asks to speak to a person, wants to make a formal complaint, mentions legal action, asks for pricing, or expresses strong negative sentiment. Capture: customer name, account number, contact phone, and a 2-sentence case summary. Then call the EscalateToTeams action which posts the case into the Sonepar Customer Service Teams channel with the conversation transcript attached. Tell the customer they will be contacted within 30 minutes during working hours (9am-6pm Mon-Fri) or first thing the next working morning.
🔗 Action to wire (next step): EscalateToTeams → a Power Automate flow that posts a Teams message into #sonepar-customer-care with the captured fields and the transcript.
7

Add Action — LookupOrderStatus

3 min · Actions tab
  1. Open the Actions tab → + Add an action.
  2. Choose New action → Create a new flow. Power Automate opens in a new tab.
  3. In Power Automate, configure the flow:
    • Trigger: When Copilot Studio calls a flow.
    • Inputs: customer_account (text), order_number (text).
    • Action: Get a row by ID — Dataverse (or Dynamics 365 Supply Chain Management). For the demo we substitute: List rows from CSV (sample) — orders-sample.csv filtered by OrderNumber eq order_number and CustomerAccount eq customer_account.
    • Respond to Copilot Studio with: status, promised_date, warehouse, carrier, tracking_link, line_count, total_value_rm, notes.
  4. Save the flow as Sonepar - Lookup Order Status.
  5. Back in Copilot Studio, the action is now available. Set the action's natural-language description so the orchestrator knows when to call it:
Call this action whenever the user has provided both a Sonepar customer account number (ACC-xxxxxx) and an order number (SO-MY-yyyy-nnnnn) and is asking about order status, ETA, tracking, or delivery details. Never call this action without both inputs.
💡 Production note: Swap the "List rows from CSV" connector for the Dynamics 365 Supply Chain Management connector, calling the Sales Order header + lines APIs, with the customer-account filter enforced server-side. The demo's CSV-based version still proves the same orchestration pattern.
8

Test, publish, and add channels

2 min · Test pane + Channels tab
  1. Open the Test your agent pane (right side). Walk through the scripted conversation:
    1. "Hi, where is my order SO-MY-2026-04221?" → agent asks for account number.
    2. Provide ACC-003122 → agent returns full order detail (status, ETA, carrier, warehouse, line count, total).
    3. "Is the ABB XT2N 160 in stock at KL?" → agent returns 42 units, reorder advised, suggests Penang as backup.
    4. "Send me the datasheet" → agent answers from the product catalog with the file reference.
    5. "I want to speak to a person" → agent collects details and posts to the Teams escalation channel.
  2. Click Publish. Wait ~30 seconds for the publish to complete.
  3. Open the Channels tab.
    • Toggle ON Microsoft Teams → click Add agent. This installs the agent into Teams for the demo tenant.
    • Toggle ON Demo website → copy the embed snippet (you can paste it into the customer portal HTML).
    • (Optional, if time) Microsoft 365 Copilot → makes the agent appear in BizChat for licensed Sonepar users.
  4. Demo the published agent inside Teams: open Teams → Apps → search "Sonepar Customer Service Agent" → run one final query end-to-end.
⚠️ Demo timing: Steps 1, 2, 7 are the only ones that take real time. Steps 3-6 are mostly Copilot Studio auto-generating from natural language — narrate while it works. Steps 7 (Power Automate flow) and 8 (publish + channel) are the highest-risk for live demo; have the flow pre-built as a backup.
